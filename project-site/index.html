DevOps Mini Project 33

Git Submodules in Multi-Repository Projects

A complete demonstration of integrating and managing external repositories inside a primary project using Git submodules.

Main Repository External Repository

Objective

Problem Scenario

A primary project depends on external libraries maintained in separate repositories. The team needs a clean and reproducible way to track these dependencies.

Solution

Use Git submodules so the main repository stores exact pointers to commits from external repositories, ensuring consistency for every developer.

Architecture

main-project/
|-- .gitmodules
|-- libs/
|   |-- external-lib/  (submodule)
|-- README.md
|-- SUBMODULE_USAGE.md
|-- PROJECT_REPORT.md
|-- demo.ps1
|-- verify-project.ps1
|-- update-submodule.ps1
|-- project-site/
    |-- index.html
    |-- styles.css
    |-- script.js

Step-by-Step Workflow

  1. Create both repositories
  2. Add external repository as a submodule
  3. Commit submodule pointer in main project
  4. Update submodule when external library changes
  5. Synchronize and verify submodule state
  6. Document all commands for team onboarding

Key Commands

Add Submodule

git submodule add <external-repo-url> libs/external-lib

Initialize on Clone

git submodule update --init --recursive

Update Submodule Pointer

git submodule update --remote --merge

Sync URLs

git submodule sync --recursive
